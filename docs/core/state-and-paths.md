## 1. Registry

Tracks installed modules and submodules in INI-style files.

| Registry | Path | Owner |
|----------|------|-------|
| System | `/etc/system-scripts/registry` | root |
| User | `~/.config/system-scripts/registry` | user |

Selection: system if `EUID=0`, user otherwise.

### Format

```ini
[clamav]
version=0.1.0
installed=2026-02-15 10:30:45

[clamav/daily-clamscan]
version=0.1.0
installed=2026-02-15 10:30:46
```

### Functions (`lib/registry.sh`)

**Module-level:**

| Function | Signature | Returns |
|----------|-----------|---------|
| `registry_set` | `(module, version)` | â€” |
| `registry_remove` | `(module)` | â€” |
| `registry_get_version` | `(module)` | version string or exit 1 |
| `registry_is_installed` | `(module)` | 0/1 |
| `registry_list` | `()` | module names |
| `registry_get_date` | `(module)` | timestamp string |

**Submodule-level:**

| Function | Signature | Returns |
|----------|-----------|---------|
| `registry_set_submodule` | `(module, sub, version [, mod_version])` | â€” |
| `registry_remove_submodule` | `(module, sub)` | auto-removes parent if last |

| Function | Signature | Returns |

## 2. Configuration files

### System-level (`/etc/system-scripts/`)

| File | Format | Generated by | Used by | Env override |
|------|--------|-------------|---------|--------------|
| `paths.conf` | Shell vars | `lib/paths.sh` (at lib install) | clamav, torrent scripts | `PATHS_CONF` || `color.conf` | Shell vars | `lib/install.sh` (from template) | `lib/color.sh` | â€” |
| `clamav.conf` | Shell vars | `shared/setup-clamav.sh` (profile-based) | clamav scripts | `CLAMAV_CONF` || `firewall.conf` | Shell arrays | `firewall-harden` | `configure-firewall.sh` | `FIREWALL_CONF` |

### State files (`/etc/system-scripts/`)

| File | Format | Created by | Used by | Lifecycle |
|------|--------|-----------|---------|-----------|
| `firewall.state` | Shell vars | `configure-firewall.sh` (first apply) | `firewall-harden restore`, `uninstall.sh` | Deleted after restore |

### User-level (`~/.config/`)

| File | Format | Module | No overwrite |
|------|--------|--------|-------------|
| `backup/system.yml` | YAML | backup/system | yes |
| `backup/hdd.yml` | YAML | backup/hdd-to-hdd | yes |
| `backup/hdd-both.yml` | YAML | backup/hdd-to-both-hdd | yes |
| `backup/bitwarden.yml` | YAML | backup/bitwarden | yes |
| `backup/vps.yml` | YAML | backup/vps | yes |
| `backup/hooks/*.yml` | YAML | backup/system | yes |
| `notify-daemon/services.conf` | Text list | notifications | dynamic |
| `notify-daemon/icons.conf` | Key=value | notifications | dynamic |
| `torrent/container.conf` | Shell vars | torrent | â€” |
| `system-scripts/bookmarks.conf` | Shell vars | nautilus | dynamic |

### Config format conventions

| Format | Used for | Pattern |
|--------|----------|---------|
| Shell-sourceable | System configs, state files | `source "$CONFIG_FILE"` |
| YAML | User configs (backup) | Parsed with `yq` |
| INI | Registry | Custom awk/grep parsing |
| Text list | Notify services | One entry per line |
| Key=value | Notify icons | `tag=icon_name` |

### Env var override pattern

All config paths support testability override:

```bash
# System configs
readonly CONFIG_FILE="${PATHS_CONF:-/etc/system-scripts/paths.conf}"
readonly FIREWALL_CONF="${FIREWALL_CONF:-/etc/system-scripts/firewall.conf}"

# User configs
readonly CONTAINER_CONF="${TORRENT_CONTAINER_CONF:-$HOME/.config/torrent/container.conf}"
readonly CONTAINER_DATA_DIR="${TORRENT_DATA_DIR:-$HOME/.config/podman/containers}"
```

### Template processing

Templates use `__PLACEHOLDER__` + `sed`:

```bash
sed -e "s|__HOME__|$real_home|g" -e "s|__USER__|$real_user|g" "$tpl" > "$dest"
```

| Placeholder | Replaced with |
|-------------|---------------|
| `__HOME__` | User home directory |
| `__USER__` | Username (from `SUDO_USER`) |
| `##READWRITE_PATHS##` | Dynamic paths (download-clamscan only) |

## 9. Format standardization summary

| Category | Standard format | Location pattern |
|----------|----------------|------------------|
| System config | Shell-sourceable (bash arrays/vars) | `/etc/system-scripts/<name>.conf` |
| System state | Shell-sourceable (bash vars) | `/etc/system-scripts/<name>.state` |
| User config (data) | YAML | `~/.config/<module>/<name>.yml` |
| User config (runtime) | Shell-sourceable (bash vars) | `~/.config/<module>/<name>.conf` |
| Registry | INI (`[section]`, `key=value`) | `/etc/system-scripts/registry` |
| Module metadata | YAML | `modules/<name>/module.yml` |
| Templates | Static with `__PLACEHOLDER__` | `modules/<name>/<type>/<file>.tpl` |
| Notify config | Text list / Key=value | `~/.config/notify-daemon/` |
| Systemd units | Standard systemd format | `/etc/systemd/system/` or `~/.config/systemd/user/` |
| Logrotate | Standard logrotate format | `/etc/logrotate.d/` |

## 10. Uninstall behavior

What is removed vs preserved when uninstalling each component.

### Global uninstall (`uninstall.sh`)

| Component | Path | Removed |
|-----------|------|---------|
| Lib directory | `/usr/local/lib/system-scripts/` | yes |
| Config directory | `/etc/system-scripts/` (all contents) | yes |
| System registry | `/etc/system-scripts/registry` | yes |
| User registry | `~/.config/system-scripts/` | yes |
| CLI symlink | `/usr/local/bin/system-tools` | yes |
| Install directory | `/opt/fedora-system-tools/` | yes |
| Gum | system binary | **preserved** (manual: `sudo dnf remove gum`) |

### Submodule engine (`lib/submodule.sh`)

The engine automatically removes: binaries, systemd units, logrotate configs,
created directories, notification registrations, and hook scripts.

Configs installed via `configs:` with `no_overwrite: true` are **preserved**.
The engine displays a yellow summary of preserved config paths at uninstall.
